# 提交指南.md

## 提交Pull-Request(PR)
**重要**
PR类型
- Bug修复: 添加bug的简要说明(如果有issue则进行关联)
- Feature: 需要是kiz标准文档确定的语法, **否则不合并**
- 库: 添加库所有要注册的函数的简易说明

**重要**
注意编码规范需基本按照kiz-code-style

    ✅ 类名大驼峰, 函数/方法/普通变量使用下划线, Private/Protect属性和局部变量使用名称后下划线命名

    ❌ 不常见的缩写

## 提交Issue

## 提交Bug报告
格式
```markdown
# 标题
===
- 简要说明
- bug复现
```

## 提交功能请求
格式
```markdown
# 标题
===
- 简介
- 所请求的相关语法
```
示例
```markdown
# 关于管道运算符的请求
===
- 管道运算符可以简化嵌套函数调用
- 所请求的相关语法:
`kiz
expression |> func_name1 |> func_name2
`
```
